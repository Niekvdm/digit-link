## Document Generation

This documentation was generated based on code review and reflects the current state of the codebase as of the documentation date. It should be updated when significant architectural changes are made.

### Updating Documentation

When making changes to digit-link, please update the relevant documentation:

1. **New API endpoints** → Update `api.md`
2. **Schema changes** → Update `database.md`
3. **Security changes** → Update `security.md`
4. **Architecture changes** → Update `architecture.md`
5. **Performance improvements** → Update `performance.md`
